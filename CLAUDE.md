This is a novel research project. Nobody has applied persistent homology to the patent citation network to detect topological precursors of technological breakthroughs. The gap in the literature is confirmed as of March 2026.

## Prior Art — What Exists and What Doesn't

**Exists:**
- Érdi et al. (2013): PageRank + clustering on USPTO citations to predict emerging tech fields
- Mariani et al. (2018): Rescaled PageRank for early identification of significant patents
- Nakamura et al. (2023): TDA Mapper algorithm on patent portfolios for firm positioning
- Various: Community detection, link prediction, citation trajectory models on patent networks

**Does NOT exist (our contribution):**
- Persistent homology (Betti numbers, persistence diagrams) computed on the patent citation graph
- Topological voids/loops/higher-dimensional signatures as precursors to breakthroughs
- Sliding-window TDA on temporal snapshots of the citation network
- Cross-CPC-class topological feature tracking over time

### Phase 2: The Topological Clock (Notebook 02) — THE NOVEL CORE

**Task 2.1: Implement src/topology.py**
This is the most important module. Take extra care.

- Function: `citation_simplicial_complex(graph, max_dim=2)` → simplicial complex from the citation graph
  - Method: Use the clique complex (flag complex) of the undirected version of the citation subgraph
  - For computational tractability, work on CPC-class-level subgraphs (not the full 8M-node graph)
  - A CPC "section pair" subgraph contains all patents in either section plus all edges between them
  
- Function: `compute_persistence(simplicial_complex, max_dim=2)` → persistence diagram
  - Use `ripser` (preferred for speed) or `giotto-tda`
  - Return: birth-death pairs for dimensions 0, 1, 2
  - β₀ = connected components (fragmentation of knowledge)
  - β₁ = 1-dimensional loops (circular knowledge flows)
  - β₂ = 2-dimensional voids (enclosed cavities — rare but meaningful)

- Function: `betti_numbers(persistence_diagram, threshold=None)` → (β₀, β₁, β₂)
  - If threshold given, count features with persistence > threshold

- Function: `persistence_entropy(persistence_diagram)` → float
  - Shannon entropy of the persistence diagram (measures topological complexity)

- Function: `sliding_window_topology(citations_df, cpc_map, cpc_pair, window_years=5, stride_years=1, max_dim=2)` → DataFrame
  - For each time window, build the CPC-pair subgraph, compute persistent homology
  - Return: year, β₀, β₁, β₂, persistence_entropy, num_long_lived_features, max_persistence

- IMPORTANT COMPUTATIONAL NOTES:
  - Persistent homology on the full patent graph is intractable. Always work on subgraphs.
  - Start with CPC section pairs (8 sections → 28 pairs). This is the primary analysis unit.
  - Within a section pair, if the subgraph exceeds ~50K nodes, subsample or use a distance-based filtration on the adjacency matrix rather than the full clique complex.
  - Use ripser's sparse input mode for large graphs.
  - Profile memory usage. A single persistence computation on a 30K-node graph can take minutes. Budget accordingly.
  - If ripser + clique complex is too expensive, fall back to:
    (a) Vietoris-Rips on a distance matrix derived from shortest-path or Jaccard distances between patents
    (b) Cubical complexes on a gridded representation of the CPC co-citation matrix
    Document which approach was used and why.

### Phase 4: The Precursor Test (Notebook 04) — THE HYPOTHESIS TEST

**Task 4.1: Implement src/nullmodel.py**
- Function: `random_cpc_pair_baseline(cpc_map, citations_df, n_samples=100, window_years=5)` → list of topology measurements
  - For each sample: pick a random CPC section pair and a random year
  - Compute the same topological metrics as we compute for real breakthroughs
  - This is the null distribution: "what does topology look like when nothing special is happening?"
  
- Function: `matched_null(breakthrough, cpc_map, citations_df, n_samples=100)` → list of topology measurements
  - More rigorous: match on CPC section pair and baseline activity level
  - Only vary the time window (shift to non-breakthrough periods for the same field pair)
  - This controls for field-specific topology

**Task 4.2: Build 04_precursor_test.ipynb**
- For each breakthrough in the catalog:
  1. Identify the relevant CPC section pair(s)
  2. Compute topological metrics (β₁, persistence entropy, max persistence, cross-class edge count) in the 10 years BEFORE the breakthrough
  3. Compute the same metrics for the matched null model (same field pair, different time windows)
  4. Test: is the pre-breakthrough topology systematically different from the null?
- Aggregate analysis across ALL breakthroughs:
  - Superposed epoch analysis (align all breakthroughs at t=0, average the topological metrics from -10 years to +5 years)
  - Compare against the null model ensemble
  - Statistical test: Kolmogorov-Smirnov or permutation test on pre-breakthrough β₁ vs. null β₁
- Key question this answers: **Do topological signatures systematically precede breakthroughs, or is the topology indistinguishable from random?**
- Visualizations:
  1. Superposed epoch plot: mean β₁ aligned at breakthrough time, with null model 95% CI band (directly parallels Memory of the Earth Notebook 5)
  2. Individual breakthrough β₁ curves for 6-9 selected breakthroughs (gallery format)
  3. Effect size distribution: for each breakthrough, the z-score of pre-breakthrough β₁ relative to null
  4. ROC-like curve: vary the "anomaly threshold" on β₁, plot hit rate vs false alarm rate
  5. If null result: make this clear and prominent. A null result is a valid finding.

### Phase 5: The Predictability Horizon (Notebook 05)

**Task 5.1: Build 05_predictability_horizon.ipynb**
- Split: train on pre-2015 breakthroughs, test on post-2015 breakthroughs
- Features per CPC section pair per year:
  - Topological: β₀, β₁, β₂, persistence entropy, max persistence, number of long-lived H₁ features
  - Simple baselines: cross-class citation count, CPC mixing rate, patent volume, mean citation age
- Target: binary — did a breakthrough occur in this CPC pair within the next 5 years?
- Models:
  - Logistic regression (interpretable baseline)
  - Random forest (nonlinear)
  - Compare: topology-only features vs. simple-only features vs. combined
- Evaluation: AUC-ROC, precision-recall, feature importance (SHAP if using RF)
- Key question: **Does persistent homology add predictive information beyond what simpler metrics provide?**
- If topology adds nothing: that's the null result. Report it honestly. It means breakthrough prediction requires information the network structure doesn't contain.
- If topology adds signal: report effect size, which topological features matter most, and at what time horizons the signal appears.
- Visualizations:
  1. ROC curves for each model variant
  2. SHAP feature importance (if topology features rank high, that's the headline)
  3. Confusion matrix for the best model
  4. Timeline overlay: model's predicted "hot zones" vs. actual breakthrough locations

## Intellectual Honesty Rules

These are non-negotiable:

1. **Report null results prominently.** If topological signatures don't precede breakthroughs, say so in the abstract, the quick results, and the README headline. Do not bury it.
2. **Cite prior work fairly.** Érdi et al. (2013), Mariani et al. (2018), and Nakamura et al. (2023) did real work on patent network analysis. Acknowledge what they found and where our contribution differs.
3. **State limitations explicitly.** Computational constraints mean we work on CPC-level subgraphs, not the full network. The clique complex is one of many possible filtrations. Different choices might yield different results.
4. **Don't oversell.** If the predictive model in NB05 has AUC of 0.55, that's barely above chance. Say that. Don't frame it as "promising."
5. **Acknowledge the breakthrough catalog is subjective.** Our choice of "breakthroughs" reflects a particular view of what counts as a major advance. Different catalogs might yield different results.
