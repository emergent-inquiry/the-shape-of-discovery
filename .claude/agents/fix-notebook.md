# fix-notebook

Debug and fix a failing notebook or source module.

## Instructions

When given an error traceback from a notebook:

1. Read the failing source file at the reported line number
2. Identify the root cause — common issues:
   - **Type errors**: ArrowStringArray vs numpy array vs pandas Series mismatches (parquet uses Arrow-backed types by default in pandas 2.0+)
   - **Memory errors**: Subgraph too large for persistent homology — reduce `max_nodes` or use sparser filtration
   - **Missing data**: Files not generated by earlier pipeline steps — check `data/` directory
   - **Import errors**: Missing packages — install with pip
   - **Performance**: Python loops over sparse matrix entries — vectorize with scipy sparse operations
3. Fix the source file (in `src/` or the notebook itself)
4. Verify the fix doesn't break existing tests: `python -m pytest tests/ -x -q`
5. Report what was changed and why

## Key Architecture Notes

- Patent IDs are strings, often stored as Arrow-backed StringDtype in pandas 2.0+
- Always normalize to numpy string arrays before building index mappings
- The topology pipeline (src/topology.py) is the most computationally sensitive code
- ripser expects dense or scipy.sparse distance matrices — not NetworkX graphs directly
- Cache files in `data/topology_cache/` are pickle format
